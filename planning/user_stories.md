# User Stories

Reference the Writing User Stories final project guide in the course portal for more information about how to complete each of the sections below.

## Outline User Roles

1. Home Cook: A user who actively creates, edits, and manages their own recipe posts to share with the community.
2. Casual Browser: A user who visits the app to discover new meals, read recipe details, and filter the feed for specific types of food.

## 2. Draft User Stories
1. **Create & Categorize**: As a Home Cook, I want to create a new recipe post and assign it a category, so that I can share my meals with the community.
2. **Customize**: As a Home Cook, I want to customize my recipe card with an icon, color, or photograph, so that my post looks good in the feed.
3. **Edit**: As a Home Cook, I want to edit my existing recipes, so that I can update the ingredients or fix mistakes.
4. **Delete**: As a Home Cook, I want to delete a recipe I previously posted, so that I can remove outdated content.
5. **Browse Feed**: As a Casual Browser, I want to scroll through a feed of recipes, so that I can find inspiration for my next meal.
6. **View Details**: As a Casual Browser, I want to click on a recipe to see the details, so that I can read the full instructions.
7. **Filter by Category**: As a Casual Browser, I want to filter the main feed by category, so that I can easily find specific types of food like snacks or dinners.
8. **View Image**: As a Casual Browser, I want to see a picture of the dish on the recipe card, so that I can decide if I want to make it.
9. **View Author**: As a Casual Browser, I want to see the author of the recipe, so that I know who created the dish.
